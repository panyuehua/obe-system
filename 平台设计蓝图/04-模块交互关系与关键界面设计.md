## 二、各角色核心操作路径

### 2.1 专业负责人工作流（最重要路径）

```
Step 1: 配置培养体系（一次性+每届更新）
  → 进入「OBE体系配置」
  → 维护培养目标 → 维护毕业要求 → 配置课程体系
  → 设置支撑关系（目标↔要求↔课程权重）
  → 运行「支撑关系检查」，修复诊断问题

Step 2: 学期中监控
  → 查看「专业达成度仪表盘」
  → 关注未达标指标点的预警红灯
  → 查阅具体课程的达成情况

Step 3: 学期末改进
  → 审阅「诊断报告」
  → 审核各课程负责人提交的改进计划
  → 查阅「AI改进建议」，决定接受/驳回
  → 生成「年度专业持续改进报告」

Step 4: 每3-5年一次专业评估
  → 拉取多届达成度对比数据
  → 参考「行业匹配度分析」更新培养目标
  → 修订新版培养方案
```

### 2.2 课程负责人工作流

```
Step 1: 学期初设置（每学期）
  → 进入「课程OBE设计」
  → 确认/调整ILO列表与权重
  → 设计考核方案（项目+权重+ILO映射）
  → 生成课程大纲并提交审核

Step 2: 学期中录入
  → 确认各次作业/考试成绩已同步
  → 关注ILO达成度实时进展

Step 3: 学期末总结
  → 查看本课程达成度报告
  → 填写「课程改进计划」
  → 提交专业负责人审核
```

### 2.3 教学质量督导工作流

```
Step 1: 发布评价问卷
  → 学期中/末发布学生评教问卷
  → 监控回收率，催收未完成问卷

Step 2: 查阅数据报告
  → 全校/分学院的达成度概览
  → 低达成课程的清单与归因分析
  → 教师教学画像的横向对比

Step 3: 出具质量报告
  → 导出专项分析报告
  → 向相关学院反馈问题清单
```
