## 二、与教务系统集成规范

### 集成方向
- **教务 → 本平台**：基础数据同步（单向推送为主）
- **本平台 → 教务**：成绩数据回写（可选双向）

### 数据项清单

| 数据类别 | 字段 | 同步频率 | 优先级 |
|----------|------|----------|--------|
| 学校/学院/专业基础信息 | 编码、名称、层级关系 | 变更时触发 | 高 |
| 教师信息 | 工号、姓名、所属学院 | 变更时触发 | 高 |
| 学生信息 | 学号、姓名、专业、年级、班级 | 学期初 | 高 |
| 课程基础信息 | 课程编号、名称、学分、课程性质 | 变更时触发 | 高 |
| 教学班信息 | 教学班编号、课程、教师、学期、学生名单 | 学期初 | 高 |
| 培养方案数据 | 专业课程计划、学分要求 | 版本更新时 | 中 |
| 学生成绩 | 学号、课程、学期、最终成绩 | 期末评分后 | 高 |

### 集成方式建议
```
优先级顺序：
1. 标准API接口（REST/WebService）—— 教务厂商提供
2. 数据库视图读取 —— 需教务系统配合开放只读视图
3. 文件导入（CSV/Excel）—— 兜底方案，手动操作
4. 中间件消息队列 —— 大规模实时同步场景
```

## 三、与教学平台（LMS）集成规范

### 集成方向
- **LMS → 本平台**：学习数据、成绩数据采集（关键数据）

### 数据项清单

| 数据类别 | 说明 | 同步频率 |
|----------|------|----------|
| 课程任务列表 | 作业、测验、考试、实验项目的基本信息 | 任务发布后 |
| 学生任务得分 | 每个学生每次任务的实际得分 | 教师评分后 |
| 任务满分 | 各任务的满分值（用于比率计算） | 任务发布时 |
| 学习行为（可选） | 资源访问次数、视频观看时长 | 日/周批次 |

### 成绩数据映射规则
```
LMS数据                      →   本平台映射
─────────────────────────────────────────────────
课程ID + 任务ID + 学生ID + 得分   →   考核项得分记录
任务ID                            →   需与考核方案中的「考核项」手动/自动匹配
满分值                            →   达成度计算基准
```

### 成绩数据关联挑战
> LMS中的任务ID与本平台考核项之间需要建立映射关系，这是集成的核心难点。
> 建议方案：在本平台配置考核方案时，提供LMS任务ID的关联字段，由课程负责人在学期初完成绑定。

## 四、外部行业数据采集规范

### 4.1 招聘平台数据接入

```
数据采集流程：

招聘平台API
（智联/猎聘/BOSS）
    │
    ▼ 定时调用（月度）
岗位数据原始库
├── 岗位名称
├── JD全文
├── 技能要求标签
├── 薪资区间
└── 发布时间

    │
    ▼ NLP清洗处理
技能标签库
├── 原始词 → 标准化词（如 "Golang" → "Go语言"）
├── 词频统计（本月出现次数）
├── 同比/环比变化
└── 行业分类归属

    │
    ▼ 向量语义对齐
校内-行业匹配结果
├── 匹配项（已覆盖）
├── 缺失项（未覆盖）
└── 溢出项（无需求）
```

### 4.2 政策文件智能体

```
触发条件：每周定期 + 手动触发

目标网站（可配置）：
- 教育部官网
- 省/市教育厅官网
- 工信部/工信厅官网
- 行业协会官网

处理流程：
1. 网络爬取 → 识别新增文件
2. PDF/Word全文提取
3. AI摘要（提取：政策要点、人才需求方向、关键技术词）
4. 生成《政策研判单》（Markdown格式，供专业负责人阅读）
5. 关键词与校内能力指标匹配评分
6. 推送至平台消息中心

注意：AI分析仅供参考，不自动修改培养方案。
```

## 五、数据权限与安全边界

```
数据分级保护策略：

等级1 - 公开数据（平台内可共享）
├── 课程基本信息
├── 专业名称与结构
└── 行业技能标签库

等级2 - 校内数据（登录可见）
├── 培养方案详情
├── 专业图谱与健康度
└── 教师教学画像（本人+督导可见）

等级3 - 专业管控数据（角色权限控制）
├── 毕业要求达成度（专业负责人/督导）
├── 学生个人达成度（本人/班主任/辅导员）
└── AI改进建议（专业负责人+学院管理员）

等级4 - 敏感数据（最高权限）
├── 原始学生成绩
├── 问卷原始回答
└── 系统配置与密钥
```

## 六、与平台的边界划分（重要）

```
本平台【不做】的事情：
✗ 不替代教务系统管理排课/选课/学籍
✗ 不替代LMS承载教学内容与互动
✗ 不直接修改教务系统中的任何数据（只读为主）
✗ 不自动修改培养方案（AI建议仅供人工审核）

本平台【专注做】的事情：
✓ 聚合两端数据，消除孤岛
✓ 按OBE逻辑计算达成度矩阵
✓ 诊断逻辑缺陷、发现质量问题
✓ 结合行业数据提供改进决策支持
✓ 生成认证所需的数据报告与材料
```
