# OBE教管一体化平台 — 核心数据模型

---

## 一、实体关系总览

```
学校(School)
  │
  ├── 办学定位(SchoolPosition)
  ├── 核心能力框架(CoreCompetencyFramework)
  │     └── 核心能力(CoreCompetency) [N]
  │
  └── 学院(College) [N]
        │
        └── 专业(Major) [N]
              │
              ├── 培养方案版本(CurriculumVersion) [多届并存]
              │     │
              │     ├── 培养目标(TrainingObjective) [N]
              │     │     └── 子目标(SubObjective) [N]
              │     │
              │     ├── 毕业要求(GraduationRequirement) [12条左右]
              │     │     └── 二级指标点(GRIndicator) [N] + 权重
              │     │
              │     ├── 课程体系(CourseSystem)
              │     │     └── 课程(Course) [N]
              │     │           ├── 认证课组归属
              │     │           └── 先修课关联
              │     │
              │     ├── 支撑关系-A: 培养目标 → 毕业要求 (强度)
              │     └── 支撑关系-B: 指标点 → 课程 (权重)
              │
              └── 教学班(TeachingClass) [学期开设]
                    │
                    ├── 关联: 课程 + 任课教师 + 学期 + 学生名单
                    │
                    ├── 课程目标ILO(ILO) [N]
                    │     ├── 关联指标点 + 支撑权重
                    │     └── 知识点(KnowledgePoint) [N] + 权重
                    │
                    ├── 考核方案(AssessmentPlan)
                    │     └── 考核项(AssessmentItem) [N]
                    │           ├── 类型(作业/测验/期末/实验)
                    │           ├── 总权重
                    │           └── ILO映射权重 [N]
                    │
                    └── 学生成绩(StudentScore)
                          └── 考核项得分(ItemScore) [N]
```

## 二、关键实体属性详表

### 2.1 专业（Major）
| 字段 | 说明 |
|------|------|
| 专业代码 | 教育部专业目录代码 |
| 专业名称 | 全称 + 简称 |
| 学制 | 年限 |
| 专业认证类型 | 工程教育/职业教育/其他 |
| 当前有效培养方案版本 | 版本号 |
| 所属学院 | 外键 |

### 2.2 毕业要求指标点（GRIndicator）
| 字段 | 说明 |
|------|------|
| 编号 | 如 R1.1, R2.3 |
| 描述 | 具体能力描述文本 |
| 所属一级毕业要求 | 外键 |
| 权重 | 在一级要求中的权重（求和=1） |
| 达成评价阈值 | 合格线（如0.60） |
| 评价方法 | 均值法/课程加权法 |

### 2.3 课程目标ILO
| 字段 | 说明 |
|------|------|
| 编号 | 如 CO1, CO2 |
| 目标描述 | 可测量的学习成果描述 |
| 所属教学班 | 外键 |
| 关联指标点 | 多对多（含支撑权重） |
| Bloom层次 | 记忆/理解/应用/分析/评价/创造 |

### 2.4 支撑关系权重配置
| 关系 | 权重含义 | 约束 |
|------|----------|------|
| 指标点 → 课程 | 该课程对指标点的贡献比例 | Σ权重可不等于1（相对强度） |
| ILO → 考核项 | 该考核项对ILO的覆盖权重 | Σ权重 = 1 |
| 考核项权重 | 该项在课程总成绩中的占比 | Σ权重 = 1 |

## 三、达成度计算数据流

```
原始数据层
┌─────────────────────────────────────────────────────┐
│  学生ID + 考核项ID + 得分 + 满分                     │
│  来源：LMS成绩 / 手动导入 / 教务系统成绩              │
└─────────────────────────────────────────────────────┘
            │
            ▼  按教学班+考核项聚合
┌─────────────────────────────────────────────────────┐
│  考核项达成度 = 班级平均分 / 满分 × 100%              │
│  （也可按个人计算后再聚合）                            │
└─────────────────────────────────────────────────────┘
            │
            ▼  × ILO映射权重
┌─────────────────────────────────────────────────────┐
│  ILO达成度 = Σ(考核项达成度 × 映射权重) / Σ权重      │
└─────────────────────────────────────────────────────┘
            │
            ▼  × 课程目标支撑权重（此处可能多课同支一指标点）
┌─────────────────────────────────────────────────────┐
│  指标点达成度 = Σ(ILO达成度 × 支撑权重) / Σ支撑权重  │
│  注：同一指标点由多门课程的多个ILO共同支撑             │
└─────────────────────────────────────────────────────┘
            │
            ▼  × 二级指标点权重
┌─────────────────────────────────────────────────────┐
│  一级毕业要求达成度 = Σ(指标点达成度 × 权重)          │
└─────────────────────────────────────────────────────┘
            │
            ▼  × 支撑强度
┌─────────────────────────────────────────────────────┐
│  培养目标达成度 = Σ(要求达成度 × 支撑强度) / Σ强度    │
└─────────────────────────────────────────────────────┘
```

## 四、行业数据模型

```
行业技能标签库(IndustrySkillTag)
├── 标签ID
├── 标签名称（原始文本）
├── 标准化名称
├── 出现频次（最近一季度）
├── 频次趋势（↑↓→）
├── 对应校内知识点ID（语义匹配后）
├── 匹配置信度（0-1）
└── 数据来源（招聘平台/政策文件/行业标准）

行业岗位需求(IndustryPosition)
├── 岗位名称
├── 技能标签列表（关联）
├── 薪资区间
├── 学历要求
├── 热度指数
└── 抓取时间

校内-行业匹配分析快照(MatchAnalysisSnapshot)
├── 快照ID
├── 专业ID
├── 生成时间
├── 溢出项列表（校内有、行业热度低）
├── 缺失项列表（行业高热度、校内未覆盖）
├── 匹配项列表
└── 总体匹配度得分（0-100）
```

## 五、图谱健康度指标模型

```
专业图谱健康度(MajorGraphHealth)
│
├── 逻辑完备性 = 有知识点支撑的指标点数 / 指标点总数
│   阈值建议：≥ 0.9 为良好
│
├── 资源饱和度 = 有资源挂载的知识点数 / 知识点总数
│   阈值建议：≥ 0.7 为良好
│
├── 任务闭环率 = 有考核任务映射的ILO数 / ILO总数
│   阈值建议：= 1.0 （必须100%）
│
├── 支撑连通性 = 有课程支撑的指标点数 / 指标点总数
│   阈值建议：= 1.0 （必须100%）
│
└── 综合健康度 = 加权计算上述四项
    权重建议：任务闭环率(40%) + 支撑连通性(30%) + 逻辑完备性(20%) + 资源饱和度(10%)
```
